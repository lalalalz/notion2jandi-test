# 알림 도메인을 개선해보자.


---


## 1. 문제점


### 1.1. 현재는 템플릿 사용이 혼재되어 있다. 

- 현재 알림 제목에서 사용될 템플릿 변수를 저장하고 있다.
    - templateVariables : 알림 제목 템플릿 변수 목록 (작성자 이름, 좋아요 누른 사람 이름)
- 현재 알림 내용에서 사용될 내용을 모두 저장하고 있다.
    - content : 알림 내용 (게시글 내용, 댓글 내용)

## 2. 템플릿을 사용하는 것이 적절한가?


### 2.1. 장단점 비교


|             | **장점**                                                                                 | **단점**                                    |
| ----------- | -------------------------------------------------------------------------------------- | ----------------------------------------- |
| **템플릿 사용**  | **렌더링시 디자인 처리가 쉽다.**

- 화면단에서 직접 문구를 조합하여 생성하기 때문에 디자인 처리를 하기가 쉽다. (예를 들어 변수에 대한 볼드처리) | **구현이 복잡하다.**

- 템플릿 관리 체계 필요
- 템플릿 처리 구현 |
|             | **통신, 저장 비용이 저렴하다.**

- 최소한의 정보만 저장하기 때문에
저장, 통신시 비용이 절감된다.                            |                                           |
| **템플릿 미사용** | **구현이 단순하다.** 

- 템플릿 관리 체계 불필요
- 템플릿 처리 구현 불필요                                        | **렌더링 시 디자인 처리가 어렵다.**                    |
|             |                                                                                        | **통신, 저장 비용이 비싸다.**                       |
